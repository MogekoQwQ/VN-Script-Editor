# VN Script Editor

一个本地优先的视觉小说脚本文本编辑器。

## 主要功能

- 结构化脚本编辑：说话人列 + 正文列
- `#1` 到 `#4` 标题导航
- 旁白、角色台词、备注行支持
- 角色映射、导出 ID、说话人颜色
- 双击进入选择模式，支持多选和 Shift 范围选择 / 取消
- 行级剪切、复制、粘贴、删除、上移、下移
- 全局撤销 / 重做
- 查找替换
- 自动保存到浏览器本地
- 项目文件导入 / 导出
- Ren'Py 片段导出
- PDF 阅读稿预览导出
- PWA 安装支持

## 本地运行

```bash
npm install
npm run dev
```

开发服务器启动后，在浏览器中打开 Vite 提供的本地地址即可。

## 构建

```bash
npm run build
```

构建产物输出到：

```text
dist/
```

## 静态部署

这是纯前端静态应用，可以直接部署到：

- GitHub Pages
- Cloudflare Pages
- Netlify
- Vercel

基础流程：

```bash
npm install
npm run build
```

然后把 `dist/` 部署到静态托管平台即可。

如果部署到 GitHub Pages 的仓库子路径，需要在 `vite.config.ts` 中设置 `base`，例如：

```ts
export default defineConfig({
  base: "/your-repo-name/"
})
```

项目代码里默认没有写死仓库名。

## 安装为桌面应用

部署到网页后，用户可以在支持 PWA 的浏览器中选择“安装应用”或“添加到桌面”。

安装后可以从以下入口启动：

- 桌面图标
- 开始菜单
- 应用启动器

它仍然是本地优先的 Web 应用，脚本文本默认保存在当前浏览器本地存储中。重要项目请定期导出 `.vns.json` 项目文件备份。

## 数据保存与备份

- 脚本文本默认保存在当前浏览器本地存储中。
- 刷新页面后，应用会优先恢复最近一次自动保存的项目。
- 清理浏览器数据、切换浏览器或使用无痕模式，可能导致自动保存丢失。
- 重要项目请使用右侧“保存”面板导出 `.vns.json` 项目文件备份。

## 项目文件说明

- 项目文件包含完整项目数据：脚本行、角色映射、设置和元信息。
- 可通过“导入项目文件”恢复编辑状态。
- 导入会覆盖当前内容，导入前会要求确认。

## Ren'Py 导出说明

- Ren'Py 导出生成的是脚本片段，不等于完整 Ren'Py 工程。
- 角色台词会根据右侧角色映射中的导出 ID 转换。
- 备注和标题可按设置导出为注释。

## PDF 阅读稿说明

- PDF 功能会打开一个可打印的阅读稿预览页。
- 在预览页顶部点击“打印 / 保存为 PDF”，再使用浏览器打印对话框保存为 PDF。
- 这更适合试读和校对，不是完整排版出版功能。

## 浏览器兼容性

- 推荐使用最新版本的 Chrome、Edge 或其他现代 Chromium 浏览器。
- 功能依赖 `localStorage`、Clipboard API、打印能力、Service Worker 和现代表单控件。

## 隐私说明

- 当前版本没有后端服务，脚本文本不会主动上传。
- 默认数据只保存在当前浏览器本地。
- 如需长期保存或跨设备继续编辑，请自行导出项目文件备份。
